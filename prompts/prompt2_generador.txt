2. INFOGRAFÍA HTML

REFERENCIA ORO: la infografía de CRDO (https://patonet.github.io/informes/equities/Informe_CRDO_107_09_190326.html) es el ESTÁNDAR VISUAL. Replicar SU ESTRUCTURA EXACTA para cada nuevo ticker — misma CSS, mismas zonas, mismos popups, mismo semáforo — solo cambiando los DATOS. Las adiciones respecto a CRDO son: (1) barra de performance histórica y (2) mini-semáforos B0.

Elementos OBLIGATORIOS que deben estar (replicar de CRDO)

-   LOGO de la empresa: img Clearbit con fallback Google Favicons, segundo fallback div con iniciales. id=triggerLogo.

-   POPUP EJECUTIVO: div#execPopup, position:fixed, centrado, fondo #0a1628, borde 2px #00d4ff. Texto del Bloque A. Se activa con mouseenter sobre triggerLogo, triggerNombre, triggerFrase. Oculta con mouseleave (delay 350ms).

-   FRASE DEFINITORIA: id=triggerFrase, cursiva entre comillas, color cian, fondo #0d1f35, border-left 3px cian. Cursor pointer. Activa el popup ejecutivo.

-   BARRA 52W: SVG con marcador rojo (min), verde (max), cian (precio actual con posición relativa %).

-   SEMÁFORO IA: SVG id=triggerIA, viewBox 0 0 80 200, width 3.5rem. Carcasa bisel #1a1a1a. 3 luces: ROJO arriba, NARANJA centro, VERDE abajo. La luz ACTIVA tiene fill brillante + glow (drop-shadow) + reflejo elíptico opacity 0.35. Las luces INACTIVAS tienen fill tenue opaco (ej: #2d0a0a para rojo apagado, #003d1a para verde apagado) + reflejo opacity 0.05. ESTO ES CRÍTICO: las luces apagadas DEBEN verse tenues, no desaparecer.

-   POPUP IA: div#iaPopup con análisis completo de la Etapa F (Bloque D). Se activa con mouseenter sobre triggerIA.

-   GAUGE CIRCULAR FEROLDI: SVG con arco gradiente rojo→verde, aguja dorada.

-   FEROLDI SCORECARD (Zona 5): cada fila con clase feroldi-row y atributo data-tooltip con descripción completa del punto. Popup #feroldiPopup al hover.

-   TABLA COMPETIDORES (Zona 6): con barra SVG de rango 52W y posición relativa % para CADA competidor.

ADICIÓN 1 — Barra de Performance Histórica

Título: “PERFORMANCE HISTÓRICO”. Datos de TradingView Scanner API. 8 periodos: 1D · 5D · 1M · 3M · 6M · YTD · 1Y · 5Y. Verde positivo, rojo negativo, gris N/D. Va en Zona 1, debajo de la barra 52W.

ADICIÓN 2 — Mini-semáforos Bloque 0

4 mini-semáforos SVG al 30% del tamaño del semáforo IA (width 1.2rem). Misma estructura (carcasa, 3 luces con activa brillante e inactivas TENUES). B0.1 Competencia · B0.2 Simplicidad · B0.3 Predictibilidad · B0 Veredicto. Etiqueta + resultado en color. Popup al hover con justificación del Bloque 0.

Gráficos Zona 3 — REGLAS ESTRICTAS

COLUMNA 1 (barras): SIEMPRE Revenue to Profit Conversion (Revenue, COGS, Gross Profit, OpExp, OpIncome, Non-Op, Taxes, Net Income). NUNCA usar barras para EPS.

COLUMNA 3 (dona/pastel): SIEMPRE composición de ingresos por segmento de negocio o geografía. NUNCA usar el pastel para otra cosa.

COLUMNA 2 (waterfall): cascada trimestral Revenue→Net Income.

Zona 1 — Debajo del precio

NO poner texto debajo del precio (delta %, YTD, vs ATH) si esos datos ya están en la barra de performance. Evitar duplicación.

Responsividad

R1: html{font-size:clamp(10px,1.15vw,17px)}. R2: SOLO rem/em/%/fr. R3-R7: mismas reglas de siempre.
