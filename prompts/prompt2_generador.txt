2. INFOGRAFÍA HTML

2.A Estilo y datos

Generar SOLO con comando INFOGRAFIA. Estrategia: const DATA = {...} con todos los valores al inicio del script. Las zonas HTML referencian DATA sin recalcular.

-   Paleta: fondo #050a12, texto #c8d8f0, cian #00d4ff, verde #00ff9d, dorado #ffd700, rojo #ff3a3a, amarillo #ffbb00. SHORT: sustituir cian por #ff1744.

-   Tipografía: monoespaciada para números, sans-serif para etiquetas. Mínimo 0.8rem en todo. @patonet en 1rem dorado.

2.B Responsividad (7 reglas obligatorias)

R1: html { font-size: clamp(10px, 1.15vw, 17px); } — 1rem escala con viewport/zoom.

R2: SOLO rem/em/%/vw/vh/fr. PROHIBIDO px excepto border 1-2px.

R3: Contenedores width:100% padding:1.5rem overflow:hidden.

R4: Grids: repeat(auto-fit, minmax(Xrem, 1fr)) o variantes fr. NUNCA px fijos.

R5: Canvas/SVG: width:100% height:auto. Chart.js: responsive:true, maintainAspectRatio:true.

R6: Gauges SVG: viewBox=“0 0 200 120” width=“100%” max-width:12rem.

R7: Verificación final: antes de cerrar </style>, todo px que no sea border → reemplazar.

2.C Estructura de 6 Zonas

ZONA 1 — Cabecera

Grid 2 columnas [logo+identidad | precio+veredicto+scores]. Logo Clearbit con fallback Google Favicons, segundo fallback div con iniciales. id=triggerLogo para popup ejecutivo.

Nombre 2rem bold blanco + sector/fecha 0.85rem #94a3b8. Popup ejecutivo: div fixed centrado, fondo #0a1628, borde 2px #00d4ff, z-index 10000, contiene Bloque A. Mouseenter sobre triggerLogo/triggerNombre/triggerFrase activa; mouseleave con delay 350ms oculta.

Frase resumen: cursiva comillas, 1rem cian, fondo #0d1f35, border-left 3px cian. Cursor pointer.

Precio 2.2rem bold + delta semafórico. Barra 52W SVG (marcadores rojo min, verde max, cian actual).

BARRA DE PERFORMANCE — CORRECCIÓN: los % de rendimiento se leen DIRECTAMENTE de StockAnalysis.com (pestaña Overview, sección Performance) o TradingView — NO se calculan desde precios de referencia. 8 periodos: 1d · 5d · 1m · 6m · YTD · 1a · 5a · Desde IPO. Embeber como const PERF_DATA = [{label,pct,ref}...]. Si no disponible: null (muestra N/D). Verde #00c853 positivo, rojo #e53935 negativo, gris #555 null.

Veredicto: 5 opciones (COMPRAR #00c853, COMPRAR EN RETROCESOS #1565C0, ACUMULAR #00838f, NEUTRAL #5d4037, NO COMPRAR #e53935). Prohibido MANTENER.

Tarjetas Feroldi: % máximo + score/160. Gauge circular SVG.

Semáforo IA: SVG 3 luces (ROJO arriba, NARANJA centro, VERDE abajo) con carcasa bisel. Luz activa con glow + reflejo 0.35; inactivas tenue + reflejo 0.05. Popup con Bloque C al hover.

ZONA 2 — Key Stats

Grid fluido de 4 subgrupos: VALORACIÓN (P/E, PEG, EPS, MktCap) · RENTABILIDAD (Márgenes, ROE, ROA) · CRECIMIENTO (Revenue/EPS Growth) · DIVIDENDO/RIESGO (Yield, Beta). Cards: valor 1.3rem mono + etiqueta 0.8rem + triángulo semafórico. Fondo #0d1828.

ZONA 3 — Gráficos financieros

CSS obligatorio: .zona3 grid 1fr 1fr 1fr gap 1.5rem. Cada .zona3-chart con .chart-wrap aspect-ratio:4/3 y canvas position:absolute inset:0. Chart.js: responsive:true, maintainAspectRatio:false.

-   COL 1 — Crecimiento y Rentabilidad: 4Q más recientes. Revenue (#1a3a6e) y Net Income (#00d4ff) comparten eje Y derecho (yVal). Margen % en eje Y izquierdo (yMg) línea naranja. Calcular límites yVal dinámicamente (soporta negativos). NUNCA min:0 si hay NI negativo.

-   COL 2 — Waterfall trimestre más reciente: barras flotantes [base,top]. Revenue→COGS→Gross→OpExp→OpIncome→NonOp→Tax→NI. Colores por naturaleza, etiquetas con valor absoluto.

-   COL 3 — Composición ingresos (dona): por segmento o geografía. Si no hay datos: estructura de márgenes. Leyenda debajo.

Hover popups en todos: fondo #1a2a4a, borde #00d4ff.

ZONA 4 — Analistas y precio 52W

Línea de precio cian. Price targets como datasets adicionales line (NO chartjs-plugin-annotation). Verde si target > precio, rojo si < precio. borderDash:[8,4]. Plugin afterDraw con guard if(!scales.y) return para etiquetas. Barra resumen: consenso 1.3rem + diferencia % + distribución calificaciones.

ZONA 5 — Perfil Feroldi

LAYOUT VERTICAL: Spider arriba (100% ancho), Scorecard debajo (grid 5 cajas). NUNCA lado a lado.

Spider SVG viewBox 600×400 width 100%. 5 ejes: Crecimiento(3,13,14) · Rentabilidad(7,11) · Solidez(6,8,12) · Estrategia(1,2,5,10) · Retorno(4,9,16). Área cian translúcida.

Scorecard: 5 cajas grid auto-fit minmax(14rem,1fr). Cada caja: header grupo + promedio dorado + fila por punto con gauge SVG semicircular (arco gradiente rojo→verde, aguja #ffd700, ángulo=180-(score/10)*180). Ids de gradiente únicos. Score 1.2rem bold dorado bajo cada gauge.

Hover popup: clase feroldi-row con data-tooltip del Bloque G. Popup fixed #0d1f35, 22rem, z-index 9999.

Pie: % máximo + score/160. 1rem bold dorado.

ZONA 6 — Riesgos, Analistas, Competidores, Veredicto

-   A. Riesgos y Catalizadores: grid 2 cols, tablas máx 6 filas c/u, ponderación Alto/Medio/Bajo con colores.

-   B. Gauge consenso analistas: SVG velocímetro grande viewBox 400×240, arco Venta Fuerte→Compra Fuerte, aguja blanca. Mini tabla: banco/calificación/target.

-   C. Tabla competidores: 5 empresas + $TICKER en fila 1. Cols: Empresa(logo) · MktCap · P/E TTM · PEG · M Bruto · M Neto · Precio · Rango 52W (barra SVG). Hover popup 10 frases comparativas. ETF: reemplazar por Top 10 Holdings + geografía + TER. SHORT: añadir Squeeze Risk.

-   D. Veredicto: badge color + tesis 3 líneas + marca @patonet 1rem dorado.

4. DIAGRAMA DEL FLUJO DE DINERO

Comando: INGRESOS. Internamente se guarda como Diagrama_Sankey_[TICKER]_[PRECIO]_[DD-MM-AAAA].html en diagramas/equities/ (o etfs/, shorts/).

HTML interactivo tipo flujo Sankey. 5 columnas: (1) FUENTES — segmentos de revenue con valor y % YoY. (2) REVENUE TOTAL. (3) CONVERSIÓN — Gross Profit (verde) + Cost of Revenue (rojo) + margen bruto %. (4) GASTOS OPERATIVOS — Op Profit (verde) + Op Expenses (rojo) + margen operativo %. (5) RESULTADO FINAL — Net Income + impuestos + no-operativos.

Conexiones: bandas SVG bezier proporcionales al valor. Verde flujos positivos, rojo costos. Hover resalta flujo completo + tooltip.

MÉTRICAS CLAVE OBLIGATORIAS EN EL DIAGRAMA: mostrar en un banner superior o lateral fijo: (1) PEG Ratio con valor numérico. (2) Score Feroldi en formato “136/160 · 85%”. (3) Veredicto en una sola palabra (COMPRAR, ACUMULAR, NEUTRAL, etc.) con color del badge correspondiente.

Estilo: fondo dark #050a12 o claro #f8f9fa según paleta. Sans-serif moderna. Responsive. Todo en español. Si falta dato: “No divulgado”.
