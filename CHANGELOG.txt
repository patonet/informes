CAMBIO ARQUITECTURAL MAYOR: REEMPLAZO DE SCRAPERS POR ALPHA VANTAGE API

MOTIVACIÓN:
  FMP free tier cubre solo S&P 100 (confirmado con pruebas reales).
  StockAnalysis y TradingView son scrapers inestables sujetos a ban.
  Alpha Vantage: NASDAQ-licensed, free tier para TODOS los tickers,
  fundamentals + ratios + analyst data en endpoints oficiales REST.
  Pruebas reales con FSLY confirmaron cobertura de small/mid caps.

[ELIMINADO] TIER 2 — StockAnalysis.com scraper
  Reemplazado por Alpha Vantage OVERVIEW + EARNINGS endpoints.
  Motivo: scraping frágil, estructura HTML cambia sin aviso, riesgo ban.

[ELIMINADO] TIER 4 — Yahoo Finance / Macrotrends fallback
  Reemplazado por Alpha Vantage. Motivo: APIs no oficiales, inestables.

[NUEVO] TIER 2 — Alpha Vantage API (oficial, NASDAQ-licensed)
  Endpoints usados (2 calls por ticker):
    OVERVIEW  → ratios, márgenes, PE/PEG/beta, 52W, analistas
    EARNINGS  → EPS histórico anual + quarterly + surprises
  AV_KEY via variable de entorno: export AV_API_KEY="tu_key"
  Rate limit: sleep(13) entre calls (5 calls/min free tier)
  Free tier: 25 calls/día → ~12 tickers/día con 2 calls cada uno
  Cobertura free: 100K+ tickers vs S&P 100 de FMP

[MODIFICADO] TIER 3 — TradingView
  Ahora solo para datos de performance (5D/1M/3M/YTD/1Y/5Y)
  y precios de competidores.
  Los demás datos (ratios, 52W, PE) ahora vienen de Alpha Vantage.

[MANTENIDO] TIER 1 — EDGAR XBRL (sin cambios funcionales)
  Sigue siendo la fuente primaria para estados financieros.
  BUG-7 fix de X0.55 activo: extract() captura FY más reciente.

NUEVOS CAMPOS EN EL JSON (antes N/D, ahora resueltos por Alpha Vantage):
  market.analyst_target_price  → precio objetivo consenso analistas
  market.analyst_buy/hold/sell → distribución de ratings
  market.analyst_strong_buy/sell
  ratios.margen_bruto          → calculado de GrossProfitTTM/RevenueTTM
  ratios.margen_op             → OperatingMarginTTM
  ratios.margen_neto           → ProfitMargin (era N/D con StockAnalysis)
  ratios.roe                   → ReturnOnEquityTTM
  ratios.roa                   → ReturnOnAssetsTTM
  ratios.ev_revenue            → EVToRevenue (nuevo)
  ratios.p_sales               → PriceToSalesRatioTTM (nuevo)
  ratios.percent_insiders      → % acciones insiders (nuevo)
  earnings.annual_eps[]        → EPS histórico 5 años
  earnings.quarterly_eps[]     → últimos 4 trimestres con surprise %

NUEVA SECCIÓN EN EL JSON: earnings{}
  annual_eps: lista de {fy, eps} últimos 5 años
  quarterly_eps: lista {fecha, reported, estimated, surprise_pct, report_date}
  Útil para Feroldi punto 3 (Historical Growth) y punto 13 (Outlook)
