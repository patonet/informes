ARQUITECTURA NUEVA:

[NUEVO] feroldi_recolectar.py
  - Recolección de datos 100% en Python async (asyncio + aiohttp)
  - Jerarquía TIER 1→4 anti-N/D con 2 rondas paralelas:
      Ronda 1: TIER 1 (EDGAR XBRL) + TIER 2 (StockAnalysis) + TIER 3 (TradingView) en paralelo
      Ronda 2: TIER 4 (Yahoo Finance / Macrotrends) solo para N/D de Ronda 1
  - Genera datos_[TICKER]_[FECHA].json con estructura unificada
  - Campo nd_resumen.campos_nd para que Claude sepa qué buscar en Ronda 3
  - Compatible con Mac (Terminal) y GitHub Actions (iPhone/cualquier dispositivo)

[NUEVO] feroldi_push.py
  - Sube archivos generados por Claude a GitHub via API
  - Auto-detección de tipo por nombre de archivo (5 patrones)
  - Actualización automática del dashboard index.html
  - Token via variable de entorno $GH_TOKEN (NUNCA hardcodeado)
  - Sin dependencias raras: solo requests (pip3 install requests)

[NUEVO] .github/workflows/feroldi.yml
  - GitHub Actions: ejecuta feroldi_recolectar.py desde cualquier dispositivo
  - workflow_dispatch con inputs: ticker + precio
  - Guarda JSON en datos/ y hace commit al repo
  - Accesible desde iPhone via github.com → Actions → Run workflow
  - Token configurado como GitHub Secret (Settings → Secrets → GH_TOKEN)

SETUP ÚNICO REQUERIDO (una sola vez):
  1. GitHub: Settings → Secrets and variables → Actions → New secret
     Name: GH_TOKEN · Value: tu_token_github
  2. Mac Terminal: echo 'export GH_TOKEN="tu_token"' >> ~/.zshrc && source ~/.zshrc
  3. pip3 install aiohttp requests beautifulsoup4
  4. Subir feroldi_recolectar.py y .github/workflows/feroldi.yml al repo

FLUJO DE USO X0.50:
  DESDE MAC:
    1. python3 feroldi_recolectar.py TICKER PRECIO
    2. Pegar JSON en Claude (o cualquier LLM)
    3. Claude corre B0 + N/D + análisis Light/Heavy
    4. Claude genera infografia_*.html localmente
    5. python3 feroldi_push.py → sube al repo

  DESDE IPHONE (o cualquier dispositivo):
    1. github.com/patonet/informes → Actions → "Feroldi Recolectar" → Run workflow
    2. Ingresar ticker + precio → ejecutar
    3. Copiar JSON desde patonet.github.io/informes/datos/datos_TICKER_FECHA.json
    4. Pegar en Claude app (iPhone)
    5. Para el push: volver a Mac con feroldi_push.py
